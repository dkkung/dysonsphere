<section class="landing__demo">
	<div class="landing__demo-code">
		```python
		import altair as alt
		import dysonsphere as ds
		from vega_datasets import data

		ds.theme()

		cars = data.cars().dropna(subset=["Miles_per_Gallon"])
		origins = ["USA", "Europe", "Japan"]

		chart = ds.mark_strip(
		    cars, "Origin", "Miles_per_Gallon", origins,
		    yTitle="Miles per gallon",
		) + ds.add_comparisons(
		    cars, "Origin", "Miles_per_Gallon",
		    [("USA", "Europe"), ("USA", "Japan")],
		    test="anova", categories=origins,
		)
		```
	</div>
	<div class="landing__demo-chart">

			annotation layer; the ANOVA, post-hoc tests, and effect sizes ride along in the export's
			metadata.

<section class="landing__features">
<CardGrid>
	<Card title="Publication defaults" icon="approve-check">
		Charts are authored at print scale with considered typography, tick geometry, and mark
		styling - the SVG that comes out is the figure that goes in the paper.
	</Card>
	<Card title="Perceptually uniform color" icon="star">
		300+ palettes built in Oklab and resampled by arc length, so sequential ramps step evenly
		and diverging ramps balance. Exportable straight to Illustrator swatches.
	</Card>
	<Card title="Statistical annotations" icon="magnifier">
		Omnibus tests, post-hoc brackets, correlation readouts - computed with scipy, drawn as
		layers you compose with <code>+</code>, reported with exact p-values and effect sizes.
	</Card>
	<Card title="Composable marks & layers" icon="add-document">
		Violins, strip plots, reference lines, text, shading, condition tables, log/power minor
		ticks - all native Altair objects that layer onto your own charts.
	</Card>
	<Card title="Self-describing exports" icon="download">
		<code>save()</code> embeds provenance, statistics, the theme, and your data into SVG, PNG,
		and JSON; <code>read()</code> and <code>load()</code> pull them back out. Every figure is
		its own lab notebook entry.
	</Card>
	<Card title="Runs in your browser" icon="rocket">
		The <a href="/playground/">playground</a> runs the real library via WebAssembly - no
		install, same output as your notebook.
	</Card>
</CardGrid>
