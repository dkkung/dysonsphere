**`dysonsphere-biology`** adds molecular-biology charts to dysonsphere - built entirely on the
public core + `ext` surfaces, so every chart it draws is themed, darkmode-aware, and
round-trips through `ds.save()` / `ds.read()` like a built-in.

```sh
pip install dysonsphere-biology
```

Once installed, its charts resolve lazily under `ds.biology`:

```python
import dysonsphere as ds

ds.biology.volcano(df, geneCol="gene", label=8)
```

## Volcano plot

`volcano()` draws log2 fold change against 鈭抣og鈧佲個 p, classifies each point as gained / lost /
non-differential by the fold-change and p-value thresholds, colors it with the diverging
`ds_div_1` endpoints (teal = gained, gold = lost), and layers on optional threshold guides and
gene labels:

<Example name="volcano" alt="Volcano plot of differential expression with labeled top genes" />

The gene labels are auto-placed with core's [`add_labels`](/guides/text-labels/#point-labels)
force-repel engine (deterministic, connector-linked), and the point ranking is domain-specific:
`label=8` selects the 8 most significant genes by the combined score `|log2fc| 脳 鈭抣og10(p)`.

### Key parameters

| Parameter | Default | What it does |
| --- | --- | --- |
| `log2fcCol` / `pvalueCol` | `"log2fc"` / `"pvalue"` | Effect-size and p-value column names |
| `geneCol` | `None` | Gene-name column (required when `label` is set) |
| `fcThreshold` | `1.0` | `|log2fc|` significance cutoff; vertical guides at 卤this |
| `pThreshold` | `0.05` | P-value cutoff; horizontal guide at 鈭抣og鈧佲個 of it |
| `label` | `None` | `int` (top-N by score), `"significant"` (all hits), or `list[str]` (named genes) |
| `thresholdLines` | `True` | Draw the fold-change / p-value guide lines |
| `palette` | `ds_div_1` ends | `(gained, lost)` hex colors |
| `nsColor` | theme grey | Non-differential point color (darkmode-aware) |
| `legend` | `True` | Show the significance color legend |

See the full signature and every parameter in the
[`dysonsphere_biology.volcano` reference](/extensions/volcano/).

:::note
Colors resolve from the active theme at call time (including the darkmode-aware grey), so for a
correct light/dark export wrap the call in a `ds.save()` callable:

```python
ds.save(lambda: ds.biology.volcano(df, geneCol="gene", label=8), "volcano")
```
:::
