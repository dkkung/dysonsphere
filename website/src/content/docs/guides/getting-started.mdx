## Install

```sh
# with pip
pip install dysonsphere

# with uv
uv pip install dysonsphere

# add as a project dependency
uv add dysonsphere
```

Requires Python >= 3.11. The runtime dependencies install with it:

- [Altair](https://altair-viz.github.io/) >= 6.0 - the charting layer
- [Polars](https://pola.rs/) >= 1.19 - the native `DataFrame` ([pandas](https://pandas.pydata.org/) input is accepted and converted)
- [NumPy](https://numpy.org/) >= 1.26 - numeric primitives
- [SciPy](https://scipy.org/) >= 1.11 - the statistics behind `add_comparisons()` / `add_correlation()`
- [vl-convert](https://github.com/vega/vl-convert) >= 1.7 - the SVG/PNG renderer behind `save()`

`pandas` and `duckdb` are optional - only needed for `read(..., output="pandas"/"duckdb")`.

dysonsphere is an [Altair](https://altair-viz.github.io/) companion. Its functions return native
Altair objects, so you keep Altair's full API and compose dysonsphere's additions with `+`. Data
can be a [polars](https://pola.rs) or a [pandas](https://pandas.pydata.org/) `DataFrame`.

## The shape of the library

Everything hangs off one namespace, `dysonsphere` (conventionally `ds`):

- **`ds.theme()`** - register a global theme. Call it once; every Altair chart afterwards inherits
  perceptually uniform palettes and publication-ready styling. See [Theming](/guides/theming/).
- **Marks** - `ds.mark_strip()`, `ds.mark_violin()` build composite plots. See [Marks](/guides/marks/).
- **Annotations** - `ds.add_rule()`, `ds.add_text()`, `ds.add_shade()`, `ds.add_labels()` return
  layers you add with `+`. See [Reference lines](/guides/reference-lines/),
  [Shading](/guides/shading/), and [Text & labels](/guides/text-labels/).
- **Statistics** - `ds.add_comparisons()` and `ds.add_correlation()` draw significance tests and
  fits that carry a report into the export. See [Comparisons](/guides/comparisons/) and
  [Correlation](/guides/correlation/).
- **`ds.save()` / `ds.read()` / `ds.load()`** - export figures that carry their own provenance,
  statistics, theme, and data. See [Saving & reading](/guides/saving/).

`ds.theme()` registers the theme; `palette="blues"` sets the categorical color scheme. Because the
theme is global, plain `alt.Chart(...)` picks it up - dysonsphere only had to set the palette.

## Saving

```python
ds.save(chart, "figure")   # writes figure.svg + figure.json by default
```

`save()` bakes provenance, the resolved theme, and (when present) statistics into the file, which
`ds.read()` and `ds.load()` pull back out later.

## Try it live

Every example on this site has an **Open in studio** link: the
[Chart Studio](/studio/) runs the real library in your browser via WebAssembly - edit the code
and watch the chart update, or upload your own data and build a chart from controls. No install
required.
