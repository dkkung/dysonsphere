`add_comparisons()` computes significance tests with scipy and draws them as a layer you compose
with `+`. Every call also builds a structured report - descriptives, exact p-values, and effect
sizes - that `save()` embeds in the export metadata, so a figure carries its own analysis. Pass
`report=True` to print it, or `save=True` to write a text file.

It classifies `test=` into **pairwise** (`mannwhitneyu`, `ttest_ind`, `ttest_rel`, `wilcoxon`,
`tukey_hsd`) or **omnibus** (`anova`, `kruskal`, `friedman`, `alexandergovern`). Here an ANOVA with
a corner label plus two post-hoc brackets. The corner label sits at the top of the plot, so pad the
y domain and lift the brackets with `yStart=` to keep everything clear:

<Example name="comparisons" alt="Fuel economy by origin with an ANOVA label and post-hoc brackets" />

## Pairwise tests and corrections

Give a list of pairs; brackets stack automatically. `correction=` applies Holm or Bonferroni
multiple-comparison adjustment.

<Example name="comparisons_pairwise" alt="Three pairwise Mann-Whitney brackets with Holm correction" />

## Compare against a control

Pass `reference=` (a group name) to compare every other group against one control and draw the
p-value **above each mark, with no brackets** - the clean many-vs-one layout that a fan of brackets
makes noisy. It derives its own comparisons (leave `pairs` unset), supports the pairwise tests, and
corrects over the whole family. Labels sit above each group's data, so overlay your points and they
clear the data; distinguishing the reference (a darker fill, etc.) stays in your hands. It also works
in grouped mode - with `xOffsetCol` set, `reference` is an xOffset level compared within each
category.

<Example name="comparisons_reference" alt="Dose-response bars compared to a control with asterisks over each bar" />

For full control you can supply the statistics and positions yourself: `pvalues=` (a dict keyed by
group) draws precomputed p-values instead of running the test, and `yPositions=` places labels - a
single number for a flat row, or a dict keyed by group to nudge individual labels.

In grouped mode a `yPositions=` dict keyed by **category** places one flat row per group - useful
when the groups span very different magnitudes and a single row can't clear them all. Here three
dose-response groups, each with the vehicle control a few shades darker, quasirandom points, error
bars, and a flat asterisk row per group compared against the control:

<Example name="comparisons_grouped_reference" alt="Three dose-response groups with per-group flat asterisk rows compared to a darker vehicle control" />

## Label styles

`labelStyle="asterisks"` renders `*` / `**` / `***` / `ns`; `bracketStyle="line"` drops the end
ticks.

<Example name="comparisons_asterisks" alt="Pairwise comparison with asterisk labels and line brackets" />

`labelStyle="value"` renders the bare number - the same as `"p"` but without the `P` symbol (to save
room), keeping a meaningful operator like `< 0.001` when the value floors. Here with `yPositions=` set
to a single number, which aligns every label in a flat row:

<Example name="comparisons_value" alt="Reference comparison with bare-value labels in a flat row" />

## Number notation

`notation=` controls the p-value format (`"scientific"`, `"e"`, `"power"`), and `sigFigs=` sets the
precision.

<Example name="comparisons_notation" alt="Comparison bracket with scientific-notation p-value" />

Both `bracketStyle` and `notation` also accept a **dict** for per-pair control (keys are pairs,
matched regardless of order); `notation` additionally takes a `"test"` key for the omnibus label:

<Example name="comparisons_perpair" alt="Two brackets with different per-pair styles and notation" />

## Omnibus with post-hoc

For an omnibus test, the corner label reports the omnibus result and the brackets are filled by the
matching post-hoc (Kruskal-Wallis → Dunn here). `omnibusVerbose=True` adds the statistic, degrees of
freedom, and effect size.

<Example name="comparisons_kruskal" alt="Kruskal-Wallis omnibus label with Dunn post-hoc brackets" />

## Grouped (two-factor) comparisons

For a grouped bar chart - a subgroup on `xOffset` within each x-category, like a qPCR gene × condition
panel - pass `xOffsetCol` to compare the subgroup **levels within each group**. `pairs` names the
levels (defaulted for two); one bracket is drawn per x-category, each with a real per-category test,
sitting above its own group's bars. Set `categories` and `xOffsetSort` to match your chart's `x` /
`xOffset` sort (it raises if they don't cover the data):
