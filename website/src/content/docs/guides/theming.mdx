`ds.theme()` registers a global Altair theme, so every chart you build afterwards inherits
perceptually uniform palettes and publication-ready styling. Call it once; override per chart with
ordinary Altair encodings. Calling it again replaces the theme entirely.

## Dark mode

Dark-background figures keep getting more common: notebooks render inline charts against the
dark editor themes most IDEs default to, and dark or black slide backgrounds are increasingly
the norm in scientific and industrial presentations alike. `ds.theme(darkmode=True)` inverts the
axis, label, and text ink while keeping palette colors intact - one flag, same chart code:

These two pin `transparent=False`, so each bakes its own contrasting `chartFill` background
(auto: white, black in darkmode). Ordinarily dysonsphere draws transparent instead, letting the
page or slide provide the ground - every chart on this site swaps ink with the site's light/dark
toggle, which is this same `darkmode` flag at work.

## Style presets

Named presets are selected with `style=`. One ships built in - `"notebook"`, a big-screen
exploration look (900 x 900 px, 18 pt type, dark, transparent) for working inline in a notebook
before you produce the print-scale figure:

```python
ds.theme(style="notebook")           # explore at screen scale
ds.theme()                           # back to publication defaults
```

A `dysonsphere.toml` config file can customise the preset or add your own named styles -
`ds.theme(style="poster")` picks up a `[poster]` section from your config. See the
[configuration guide](/guides/configuration/) for the full file format, search paths, and merge
order; `ds.create_config()` scaffolds a starter file.

## Frame and grid

`closed=True` draws a full frame around the plot (all four spines). Closed plots also give
the data a minimal inset from the frame (`viewPadding`, on by default) so marks at the domain
edge don't sit on the border - pass `viewPadding=False` for a flush frame, or a pixel value
for more room (Vega-Lite rounds any request up to the next nice tick step):

<Example name="closed_frame" alt="Scatter plot with a closed frame" />

`grid=True` adds grid lines - solid by default; `dashedGrid=True` dashes them:

<Example name="grid_lines" alt="Line chart with grid lines" />

## Rounded marks

`cornerRadius=True` rounds bar tips (and rects and arcs) by a size derived from the chart
dimensions; pass a float for explicit pixels.

<Example name="corner_radius" alt="Bar chart with rounded bar tips" />

## Label angles

`xLabelAngle` / `yLabelAngle` rotate axis labels (in degrees); alignment is derived from the sign,
so crowded categories stay legible.

<Example name="label_angle" alt="Bar chart with angled x-axis labels" />

## Inward ticks

`inwardTicks=True` points tick marks into the plot area (physics-journal style) and defaults the
frame to closed:

## More knobs

The theme exposes dozens of parameters - `axisOffset`, `tickSize`, `inwardTicks`, `fontSize` /
`secondaryFontSize` / `smallestFontSize`, `markSize`, `markStrokeWidth`, `bandPadding`, `sigFigs`,
and the axis-element toggles (`xAxis`, `xTicks`, `xLabels`, and their `y` counterparts). The
[configuration guide's key reference](/guides/configuration/#key-reference) documents every one
of them, with defaults - and each is equally valid as a `theme()` keyword or a `dysonsphere.toml`
entry.
